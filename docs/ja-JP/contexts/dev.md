# 开发上下文 (Development Context)

模式：活跃开发 (Active Development)
核心：实现、编码、功能构建

## 行为规范 (Behaviors)
- 先写代码，后做说明
- 优先考虑可运行的方案，而非完美的方案
- 变更后运行测试
- 保持提交（Commit）原子化

## 优先级 (Priorities)
1. 先跑通 (Make it work)
2. 做正确 (Make it right)
3. 做整洁 (Make it clean)

## 推荐工具 (Recommended Tools)
- 代码变更：使用 Edit, Write
- 测试/构建运行：使用 Bash
- 代码搜索：使用 Grep, Glob
