# /learn - 提取可重用模式

分析当前会话（Session），并提取具有作为技能（Skill）保存价值的模式（Pattern）。

## 触发时机

在解决非显而易见的问题时，可在会话中的任何时间点运行 `/learn`。

## 提取内容

关注以下内容：

1. **错误解决模式（Error resolution patterns）**
   - 发生了什么样的错误
   - 根本原因是什么
   - 进行了哪些修复
   - 是否可针对类似错误重用

2. **调试技术（Debugging techniques）**
   - 非显而易见的调试步骤
   - 配合良好的工具组合
   - 诊断模式

3. **规避方案（Workarounds）**
   - 库的特殊特性或缺陷（Quirks）
   - API 限制
   - 特定版本的修复

4. **项目特定模式（Project-specific patterns）**
   - 发现的代码库规约
   - 所做的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[模式名称].md` 中创建技能文件：

```markdown
# [描述性的模式名称]

**提取日期:** [日期]
**上下文:** [何时适用的简要说明]

## 问题
[要解决的问题 - 需具体]

## 解决方案
[模式/技术/规避方案]

## 示例
[如果适用，请提供代码示例]

## 使用时机
[触发条件 - 应当启用此技能的情况]
```

## 流程

1. 审查会话中可提取的模式
2. 识别最有价值/可重用的见解（Insights）
3. 草拟技能文件
4. 在保存前征得用户确认
5. 保存至 `~/.claude/skills/learned/`

## 注意事项

- 不提取琐碎的修复（如拼写错误、简单的语法错误）
- 不提取一次性问题（如特定 API 的临时故障）
- 专注于能在未来会话中节省时间的模式
- 保持技能专注 —— 一个技能对应一个模式
