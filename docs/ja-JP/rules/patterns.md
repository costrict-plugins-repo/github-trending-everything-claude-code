# 通用模式

## 骨架项目（Skeleton Project）

在实现新功能时：
1. 搜索经过实践验证的骨架项目
2. 使用并行智能体（Agent）评估选项：
   - 安全评估
   - 可扩展性分析
   - 相关性评分
   - 实现计划
3. 克隆最佳选项作为基础
4. 在已验证的结构内进行迭代

## 设计模式

### 仓库模式（Repository Pattern）

在一致的接口背后封装数据访问：
- 定义标准操作：findAll、findById、create、update、delete
- 具体实现处理存储详情（数据库、API、文件等）
- 业务逻辑依赖于抽象接口，而非具体的存储机制
- 允许轻松更换数据源，并简化基于 Mock 的测试

### API 响应格式

所有 API 响应使用一致的外壳（Envelope）：
- 包含成功/状态指示符
- 包含数据负载（错误时为 null）
- 包含错误消息字段（成功时为 null）
- 在分页响应中包含元数据（total、page、limit）
