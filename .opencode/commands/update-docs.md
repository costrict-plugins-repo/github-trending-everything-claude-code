# 更新文档命令 (Update Docs Command)

根据最近的更改更新文档：$ARGUMENTS

## 你的任务

1. **识别已更改的代码** - `git diff --name-only`
2. **查找相关文档** - README、API 文档、指南
3. **更新文档** - 保持与代码同步
4. **验证准确性** - 确保文档与实现一致

## 文档类型

### README.md
- 安装说明
- 快速入门指南
- 功能概览
- 配置选项

### API 文档 (API Documentation)
- 端点（Endpoint）描述
- 请求/响应格式
- 身份验证详情
- 错误代码

### 代码注释
- 公共 API 的 JSDoc
- 复杂逻辑解释
- TODO/FIXME 清理

### 指南 (Guides)
- 操作教程 (How-to tutorials)
- 架构决策记录 (ADRs)
- 故障排除指南

## 更新自检清单

- [ ] README 反映了当前功能
- [ ] API 文档与端点匹配
- [ ] 已为更改的函数更新 JSDoc
- [ ] 示例运行正常
- [ ] 链接有效
- [ ] 版本号已更新

## 文档质量

### 优质文档
- 准确且最新
- 清晰且简洁
- 包含可运行的示例
- 覆盖边缘情况

### 避免
- 过时的信息
- 缺失参数
- 错误的示例
- 含糊不清的语言

---

**重要提示**：文档应与代码更改同步更新，而不是事后补课。
